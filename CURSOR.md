# Cursor에서의 하네스 엔지니어링

안드레아 카파시가 말하는 **하네스**는 모델 가중치가 아니라 그 주변이다: **무엇을 읽게 할지(맥락)**, **무엇으로 행동하게 할지(도구)**, **언제 멈추고 고칠지(검증 루프)**. Cursor는 그걸 규칙·에이전트·터미널·MCP로 조립하는 IDE다. 이 문서는 이 저장소에서 그 조립을 **얇고 재현 가능하게** 유지하는 방법을 정리한다.

## 문서 역할 (중복을 피하기)

| 파일 | 역할 |
|------|------|
| `.cursorrules` | 에이전트에 항상 올라가는 **짧은 하네스 계약** → `docs/coding-rules.md`로 분기 |
| `docs/coding-rules.md` | 코딩 규칙 **진입점·목차** (영역별 `.mdc` / `docs` 링크) |
| `CLAUDE.md` | 코딩 **태도**: 가정·단순성·수술적 diff·완료 조건. |
| `CURSOR.md` (이 파일) | Cursor **제품 기능**을 하네스 층으로 쓰는 실무 가이드. |

같은 말을 세 파일에 붙여 넣지 않는다. 늘릴 때는 **한 곳**을 정해 옮기거나 링크한다.

---

## 하네스 층 (Cursor 기준)

1. **맥락:** `@` 참조, 열린 파일, 검색으로 *관련 코드*를 먼저 붙인다. 추측으로 채우지 않는다.
2. **도구:** 터미널·린터·테스트·(있다면) MCP는 **측정**용이다. “한 번 잘 나왔다”가 아니라 **같은 명령이 같은 결과**를 내게 한다.
3. **루프:** 작은 변경 → 검증 → 실패 시 원인 좁히기. 완료 조건은 `CLAUDE.md`의 “목표가 곧 완료 조건”과 같이 둔다.

---

## 설계 규칙

- **얇게:** 규칙·프롬프트·훅은 “에이전트가 자주 틀리는 한 가지”를 막을 때만 추가한다. 방어 문구만 늘리는 건 하네스가 아니라 **노이즈**다.
- **게이트 하나:** CI·스크립트·`npm test` 등 **대표 한 줄**을 정해 두면, 그 줄이 하네스의 공통 출구다.
- **재현:** 샘플링·RAG·에이전트 경로가 있으면 시드, 모델/버전, 입력 스냅샷을 남길 수 있게 한다. 디버깅은 로그 없이 하지 않는다.
- **회귀 먼저:** 규칙이나 프롬프트를 바꿀 때는 “이전에 통과하던 것이 무엇인지”를 먼저 적는다.

---

## 피할 것

- 검증 없이 `.cursorrules` / 규칙만 장편으로 늘리기.
- MCP·에이전트를 **맥락 대용**으로 쓰기 (도구는 보조, 읽을 파일이 본론).
- 이 저장소 밖 비밀·키를 규칙이나 채팅에 박아 넣기.

## 재사용 프롬프트 (복사해서 채팅에 붙여넣기)

규칙 파일: [`.cursor/rules/react-usestate-object.mdc`](.cursor/rules/react-usestate-object.mdc) — `frontend/**/*.tsx` 작업 시 자동 제안된다.

### React — 여러 `useState`를 객체 하나로 압축

아래 참조 코드를 기준으로, 이 파일(또는 지정한 컴포넌트)의 **여러 `useState`를 `useState` 객체 하나**로 바꿔 줘. `input`에는 `name`을 맞추고, `onChange`로 객체를 갱신해. 타입(`FormState`)도 정의해. 요청 범위 밖 로직은 건드리지 마.

**참조:**

```tsx
const handleSignup = async (e: React.FormEvent<HTMLFormElement>) => {
  e.preventDefault();
  const formData = new FormData(e.currentTarget);
  const formProps = Object.fromEntries(formData.entries());
  // ...
};
```

**한 줄 버전:**

```
다음 FormData 패턴을 참고해서, 여러 개의 useState를 useState 객체 하나로 압축해 줘. input name/onChange와 FormState 타입까지 맞춰 줘.
```

## 요약

모델은 한 조각이고, **Cursor + 이 저장소의 규칙·테스트·스크립트**가 나머지다. 그 조각을 **짧게 유지**하고, **맥락과 검증**으로 품질을 낸다.
