# com.ragwatson

**Arcana** 에이전트 파이프라인(문서 입력 → Hermes → Chiron → Thoth → Minerva → 출력)을 설계·실험하는 작업 공간이다. 이름은 RAG·추론 계층을 **왓슨식 조수**가 아니라 **측정 가능한 하네스**로 두겠다는 취지에서 붙였다.

## 왜 “하네스”인가

안드레아 카파시가 강조하는 것과 같다: 품질의 대부분은 가중치 한 덩어리가 아니라 **맥락(무엇을 읽게 하는지)**, **도구(무엇으로 행동하게 하는지)**, **검증 루프(언제 고칠지)**에서 나온다. 이 저장소는 그 세 층을 **얇게** 유지하고, “한 번 잘 나왔다”가 아니라 **재현·게이트**로 끝내는 방향을 문서와 규칙에 적어 둔다.

## 저장소 뼈대

- **`docs/`** — Arcana 설계·Obsidian 메모. 에이전트 시스템의 “맥락 사전” 역할.
- **`agora/nexus.proteier/arcana/app/`** — Hermes, Chiron, Thoth, Minerva 등 **앱 층 스텁·실험 코드** (`thoth.py` 등).

앱 코드가 커지면 **대표 검증 한 줄**(예: 테스트 또는 스크립트)을 README나 `CONTRIBUTING`에 적어 두는 것이 하네스를 고정하는 가장 싼 방법이다.

## 요약

모델 이름과 파이프라인 다이어그램만으로는 부족하고, **무엇을 넣었을 때 어떤 검증을 통과하면 머지인지**가 하네스다. 그 정의는 코드·CI·문서 중 한 경로로라도 **실행 가능**하게 두는 것을 목표로 한다.
